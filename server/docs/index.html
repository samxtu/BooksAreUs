    Introduction

    Welcome to the Books Are Us API!

    Books Are Us is a library application that allows users borrow books and read books. With this guide, you'd learn how to access our API endpoints in order to perform operations like account creation, login, querying library and checkout out book information, borrow books, return them, contribute books to the database and more.

    We have language bindings in Javascript! You can view code examples in the dark area to the right.

    All api routes must be prepended with /api/v1 where v1 represents the api version number.

    Authentication

    Books Are Us requires authentication with username and password for access to protected api routes. On signup and subsequent logins, a token is generated and required for requests to protected routes.

    The token is expected to be provided in the x-access-token-header request header.

    Errors

    The Books Are Us api uses the following error codes

    Error Code Meaning
    400 Bad Request -- Invalid request syntax.
    401 Unauthorized -- Failed authorization.
    403 Forbidden -- The request is not allowed.
    404 Not Found -- The specified resource could not be found.
    409 Conflict -- Your request has a contains conflicting data.
    422 Unprocessable Entity -- The request could not be processed with the data provided
    500 Internal Server Error -- We had a problem with our server. Try again later.
